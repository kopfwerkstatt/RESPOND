About the RESPOND Framework

This website is a live demo that introduces the different technologies and advantages of the Responsive Web Design + Server Side Component (RESS) Framework. By using the requesting browser's User Agent information, a responsive website with device specific partials that match the browser's features is rendered server-side. The Framework uses a server-side Device Detection library to extract the device information from the User Agent. If the browser supports JavaScript and Cookies, the detected information is compared with a client-side device and resolution-check. This compared information is stored in a Cookie and is updated if the size of a window or the orientation of a device is changed. Based on the stored information, device specific partials and resolution optimized images can be delivered.

The Framework uses three main techniques, based on the detected device information:

  1. Real Responsive Images: The Framework delivers device appropriate re-scaled versions of your images, based on the detected resolution. A device with a small screen will get a smaller image, a device with a larger screen gets larger images. See the Examples below.
  2. Device Specific Partials: Different page components are only required on a mobile site and others on it's desktop version. This Website uses the Mustache Template Engine to define and deliver different partials for different device classes.
  3. Minify CSS and JavaScript: Minify your defined CSS and JavaScript files automatically by simply calling the Minifier page of the Framework.

Try to resize the window and see how it all works! You will realize that the website is reloaded at two breakpoints. This breakpoints simulate the transition from desktop to tablet and from tablet to mobile view. At this points, the device specific partials are switched. This automated reloading is only required to simulate and test the tablet and mobile appearance on a desktop monitor. On mobile devices, only the device specific partials are loaded and no automated reloading is triggered.

The Framework delivers device appropriate re-scaled versions of your images, based on the detected resolution. A device with a small screen will get a smaller image, a device with a larger screen gets larger images. You can see the current resolution below the image.

You don't have to remember the paths to your images anymore. Use the following simple Mustache snippet to pass the file name to the Server. The Server detects the resolution of the screen and looks for a matching image. If there is one, the path to the image is returned. If not, the large version of the image is re-scaled, optimized and stored in the appropriate folder.

Device Specific Partials

The Framework detects the accessing device and includes the matching partials into your layout. With this approach, it's possible to switch the layout on any device. You think some elements of your site should be optimized or are not needed on a mobile device? No Problem! If the user likes to see the desktop partials on his Smartphone, he can simply switch the version and gets the same content as he would access your website with his notebook or desktop pc.


Try to resize the window and watch the RESPOND Logo and the Carousel at the Top of the page. This elements are implemented with partials and are switched, based on the accessing device.
